## How this is organised

Five domains, ordered by how an institutional product engineer reads a component library.

**Foundations** are the system's physical laws — tokens, type, spacing, motion, contrast.
Change a foundation and the rest flexes. Don't change them without writing the migration first.

**Primitives** are single-purpose form atoms (Button, Input, Toggle).
They have to be boring and correct. Every product surface in the firm uses them.

**Components** are assembled UI for a specific job (Card, Alert, Tabs, Modal, Spinner).
Composed from primitives + tokens. This is where product personality emerges.

**Financial** is institutional craft — a PnL cell that aligns at the decimal point,
an order row that fits 11 columns at fixed pixel widths, a regime chip that reads
at 11px in tabular-nums. These exist because the generic versions in other systems
lose meaning at a trading desk.

**Compliance** encodes a regulatory workflow rather than a UI shape.
FIX State Chip cites SEC Rule 17a-4. Threshold Band cites MAS Notice 626 §8.3.
KYC Step cites FATF Rec 10. Each docs page surfaces the citation.

**Patterns** sit on top — proven compositions (Command Palette, Dashboard Grid)
the team can reuse rather than re-derive.

---

## Conventions

- **Vanilla JS · HTML-first.** No React, no JSX, no framework lock-in.
- **Token-driven.** Every colour, spacing, type, motion, elevation value is a CSS custom property.
- **BEM with `.eds-` namespace.** No CSS-in-JS. Reads like English, greppable, survives a CSS reset.
- **Tabular-nums on every numeric cell.** PnL columns read top-to-bottom, not left-to-right.
- **Direction is never colour-only.** Up / down / blocked / cleared always pair colour with arrow + sign + label.
- **WCAG 2.1 AA is the floor.** The a11y addon runs on every story.
- **Live data does not animate.** Price ticks update instantly; perceived-latency cost is real on a desk.
- **Two themes.** Institutional Dark (default) + Salt Light (Xanthos / Christie's / cream surfaces).
- **Two density modes.** Comfortable (default) + Compact (institutional dense).

## Getting started

```bash
git clone https://github.com/Edwson/Ed_Storybook
cd Ed_Storybook
nvm use
npm install
npm run storybook        # → http://localhost:6006
npm run build-storybook  # → ./storybook-static
```

Production builds auto-deploy from `main` via GitHub Actions to
[edwson.github.io/Ed_Storybook/](https://edwson.github.io/Ed_Storybook/).
